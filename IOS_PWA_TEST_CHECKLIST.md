# iOS PWA 테스트 체크리스트

## 1) iPhone Safari 기본 동작
- `https://<도메인>/` 접속 후 Builder 화면이 깨지지 않고 스크롤/입력이 가능한지 확인합니다.
- `/i/<slug>` 초대장 뷰와 `/admin` 관리자 화면이 각각 정상 로딩되는지 확인합니다.
- 입력 폼 포커스 시 화면이 과도하게 확대되지 않는지 확인합니다.
- 하단 홈 인디케이터/상단 노치 영역에서 버튼과 텍스트가 잘리지 않는지 확인합니다.

## 2) PWA 설치(Add to Home Screen)
- Safari 공유 메뉴에서 `홈 화면에 추가`를 실행합니다.
- 홈 화면 아이콘이 정상 표시되는지 확인합니다.
- 앱 아이콘으로 실행 시 브라우저 UI 없이 standalone 형태로 열리는지 확인합니다.

## 3) 서비스워커/오프라인 기본 검증
- 최초 1회 온라인 접속 후 앱을 종료했다가 다시 열어 정적 화면이 빠르게 열리는지 확인합니다.
- 네트워크를 끈 상태에서 정적 자원(CSS/JS/아이콘)이 캐시로 표시되는지 확인합니다.
- 네트워크를 끈 상태에서 `/api/*` 요청은 실패해야 하며, 앱이 오류 메시지를 표시하는지 확인합니다.

## 4) 초대장 핵심 플로우
- Builder에서 저장 후 생성된 짧은 링크(`/i/<slug>`)가 정상 동작하는지 확인합니다.
- 초대장에서 RSVP 제출 성공/실패 메시지가 정확히 표시되는지 확인합니다.
- Admin에서 목록/상세 RSVP 조회가 기존과 동일하게 동작하는지 확인합니다.

## 5) 회귀 점검
- 기존 URL 규칙(`/`, `/admin`, `/i/<slug>`, `/api/*`)이 모두 유지되는지 확인합니다.
- 카카오/네이버/구글 지도 링크 버튼이 정상 작동하는지 확인합니다.
